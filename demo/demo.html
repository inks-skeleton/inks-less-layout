欢迎使用LayoutSimple简易CSS布局框架

LayoutSimple是一款简易的自由度极高的前端响应式布局CSS框架，项目使用LESS开发，含有一些实用的LESS Mixin，如果对LESS有些功底的朋友可以自由发挥和组合。
项目地址：https://github.com/qc-web-y/LessTool
适用人群：前端开发布局随意性较大，自由度需要很高的前端开发项目
● 起步 ●
小主，您还在辛苦的初始化布局css吗，这里帮您全自动配置好了哟！
  • 方法一：自由度较高，需要掌握有less功底
  • 1.下载项目中的src文件，在您的项目中给他建一个窝（文件夹），将它复制拷贝过去
  • 2.建议使用vscode开发，并安装插件easy less，这样项目中每个less头部的注释可以很好的利用起来，具体的请参考easy less文档介绍
  • 3.看下下面的文档，几分钟就够了，你会很快熟悉起来，如果不想熟悉我的命名方式，没关系，你可以自己到组件component中尽情的diy一个你喜欢的，保存下来，之后就可以一直用这套极简快速布局框架了
  • 方法二：自由度中等，需要掌握一些less基础，这个可以速成，详见我的博客https://www.cnblogs.com/leona-d/category/940013.html
  • 1.下载项目中的src文件
  • 2.打开config中的_default.less根据config/_default.less进行配置
  • 3.编译LayoutSimple.less文件（这里推荐使用vscode打开项目并安装easy less进行编译）
  • 4.编译成功，将生成的LayoutSimple.css 或 LayoutSimple.min.css放入自己的项目中直接使用即可
  • 方法三：自由度一般，仅需掌握css即可
  • 直接下载dist中的LayoutSimple.css 或 LayoutSimple.min.css放入自己的项目中直接使用
● config/_default.less默认配置 ●
LayoutSimple在src/config/_default.less提供一个默认配置的less文件
  • @default_fs: 初始化时默认字体大小
  • @default_fs_mobile: 初始化时默认在移动设备上的字体大小，这里为了适应移动端，960以下的字体将会使用该项
  • @default_fc: 初始化时默认字体颜色
  • @default_ffamily: 初始化时默认字体类型
  • @default_bg: 默认body背景，可以为图片或者颜色，与css写法一致
  • @default_w: body,html的宽度
  • @default_min_w: body,html最小宽度
  • @default_max_w: body,html最大宽度
  • @default_h: body,html高度
  • @default_linkC: a标签伪类:link的颜色
  • @default_visitedC: a标签伪类:visited的颜色
  • @default_selected: a标签伪类:hover,:active,:focus的样式
  • @screen_xl: .col-xl- 时需要兼容的屏幕尺寸
  • @screen_lg: .col-lg- 时需要兼容的屏幕尺寸
  • @screen_md: .col-md- 时需要兼容的屏幕尺寸
  • @screen_sm: .col-sm- 时需要兼容的屏幕尺寸
  • @screen_xs: .col-xs- 时需要兼容的屏幕尺寸
  • @title_start: h6~h1是LESS循环生成的，因此需要一个起始值，即h6的字体大小
  • @title_start_mobile: 和@title_start一致，即h6移动端的字体大小
  • @title_spacing: 前面说了，h6~h1是LESS循环生成的，因此这个就是其之间每次递增的数值，比如@title_start:16,@title_spacing:6那么h6的字体大小将为16px，h5的将为22px，h4的将为28px，依次递增。移动端也是相同的原理
  • @full_scale: LayoutSimple移动端使用rem字体，这是为移动端html根字体适配使用的，即满屏比例，比如你的设计稿是720px的，那满屏比例为720/100，设计稿是640px的，满屏比例为640/100
  • @fit_spacing: 适配间距，即每个多少px进行一次媒体查询的适配
  • @fit_range: 适配范围，由于PC对rem的支持性不高，因此仅对960-360之间的移动端进行适配，当然你可以根据需求自行配置
● 网格布局 ●
与Bootstrap4基本一致，不知道请自行了解，这里不多介绍！
  • .col- : 针对所有设备
  • .col-lg- : 默认针对960px以下的屏幕
  • .col-md- : 默认针对720px以下的屏幕
  • .col-sm- : 默认针对540px以下的屏幕
  • .col-xs- : 默认针对360px以下的屏幕
  • .hidden : 所有设备隐藏
  • .hidden-lg, .hidden-md, .hidden-sm, .hidden-xs 针对不同屏幕尺寸
  • .visible : 所有设备显示为block
  • .visible-ib : 所有设备显示为inline-block
  • .visible-inline : 所有设备显示为inline
  • .visible-lg, .visible-ib-lg, .visible-inline-lg 针对不同屏幕尺寸

当然，你也可以自行定制属于你的网格布局，这里在config/_default.less中去进行配置

如果配置中的设置仍然无法满足你的需求，且你对less也所掌握，请移步src/component/_column.less去尽情diy你的网格布局吧！

● 文字排版 ●
提供一些简单与Bootstrap相似的排版
  • h1~h6按照config/_default.less中的配置进行循环生成，这里说下循环计算标准：从h6开始，字体依次往上递增6，padding为当前字体大小-10,line-height为当前字体大小*1.6
  • .txt-samll: 12px,移动端为0.24rem
  • .txt-mini: 10px,移动端为0.22rem
  • .txt-blod: 字体加粗 font-weight: bold;
  • .txt-normal: 字体正常 font-weight: normal;
  • .txt-light: 字体超细font-weight: lighter;
  • .txt-l: 字体居左
  • .txt-c: 字体居中
  • .txt-r: 字体居右
  • .txt-ellipsis: 文本超出省略号显示
  • .txt-nowrap: 强制文本不换行

h1~h6的相关配置请参考config/_default.less默认配置

如果配置中的设置仍然无法满足你的需求，且你对less也所掌握，请移步src/component/_column.less去尽情diy你的网格布局吧！

● 简单布局 ●
简单一些提供前端常用的css布局代码
  • .jumbotron: 超大屏幕，会给左右增加一个20px的paddding，移动端为0.2rem
  • .container: 主要内容容器，所有设备都会加上一个1200px的宽度限制，并居中，1200以下的内容会自动切换为宽度100%，左右20px的padding，移动端为0.2rem
  • .ly-box: 怪异盒子模型，不懂的请自行百度
  • .ly-top-ib: 内联块级垂直靠顶部
  • .ly-fl: 左浮动
  • .ly-fr: 右浮动
  • .ly-wh100: 宽度均为100%
  • .ly-wh100-b: 宽度均为100%的块级元素
  • .img-responsive: 宽高自动，最大宽高不超过父元素的自适应图片
  • .img-responsive-w100: 高自动，最大宽不超过父元素的自适应图片
  • .img-responsive-h100: 宽自动，最大高不超过父元素的自适应图片
